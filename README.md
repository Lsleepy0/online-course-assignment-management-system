# 在线课程作业管理系统

基于 FastAPI + Jinja2 + Bootstrap 5 + SQLite 构建的在线课程作业管理系统，支持管理员、教师、学生三种角色，涵盖班级管理、用户管理、课程管理、题库管理、作业管理等完整功能模块。

> 本项目为软件测试课程的被测对象，系统中包含若干故意植入的缺陷供测试实践使用。

## 技术栈

- **后端**：FastAPI 0.104 + SQLAlchemy 2.0 + SQLite
- **前端**：Jinja2 模板 + Bootstrap 5 CDN + 原生 JavaScript
- **认证**：JWT (python-jose) + bcrypt 密码哈希
- **验证码**：Pillow 生成数字图片验证码
- **服务器**：Uvicorn

## 功能模块

### 1. 用户注册与登录模块
- 登录（用户名 + 密码 + 数字图片验证码）
- 注册（角色选择、班级选择）
- 修改密码
- 退出登录
- 根据角色自动跳转不同首页

### 2. 班级管理模块（仅管理员）
- 三级层级组织树（分部 → 专业 → 班级）
- 左键展开/折叠、右键菜单（增删改）
- 节点详情展示

### 3. 用户管理模块（仅管理员）
- 用户列表分页展示
- 多条件组合查询
- 新增/修改用户（基本信息 + 扩展信息）
- 删除用户

### 4. 课程管理模块（教师、管理员）
- 课程 CRUD
- 课程关联班级

### 5. 课程题库管理模块
- 试题录入（单选/多选/判断/填空/简答）
- 试题审核流程（未审核 → 待审核 → 已发布 → 已关闭）
- 题库管理（搜索、关闭、删除）

### 6. 课程作业管理模块
- 作业创建与发布（教师/管理员）
- 作业批改（客观题自动批改）
- 学生答题与提交
- 历史作业查看

## 快速开始

### 环境要求
- Python 3.9+

### 安装与运行

```bash
# 1. 安装依赖
pip install -r requirements.txt

# 2. 启动服务（首次启动会自动初始化数据库并写入预置数据）
python -m uvicorn main:app --host 0.0.0.0 --port 8000

# 后台运行
nohup python -m uvicorn main:app --host 0.0.0.0 --port 8000 &
```

访问 http://localhost:8000 即可使用系统。

## 技术特性

- **JWT 会话管理**：登录后通过 Cookie 存储 JWT token，有效期 12 小时
- **bcrypt 密码加密**：所有用户密码均以 bcrypt 哈希存储
- **图片验证码**：使用 Pillow 动态生成 4 位数字验证码
- **后端分页**：列表接口支持分页查询，前端显示页码导航
- **三级组织树**：班级管理支持分部 → 专业 → 班级三级层级结构
- **作业自动批改**：客观题（单选/多选/判断）提交后自动判分

## 故意植入的缺陷清单（软件测试作业）

本系统作为软件测试课程的被测对象，共故意植入 **25 个缺陷**，分布在 6 个功能模块中。缺陷均为真实的代码逻辑问题，可通过正常的 UI 操作触发。测试人员需通过黑盒测试发现并报告这些缺陷。

> 缺陷分布：题库管理模块最多（7 个），与该模块业务复杂度最高一致。

### 班级管理模块（4 个）

| 编号 | 严重等级 | 缺陷描述 |
|------|----------|----------|
| BUG-001 | 严重 | 新建节点名称超过 30 字符时，后端不做长度校验，直接存入数据库，前端也不拦截，保存成功无错误提示 |
| BUG-002 | 中 | 修改节点名称时同样不做 30 字符长度校验，保存成功无错误提示 |
| BUG-003 | 中 | 删除有子节点的节点时，后端不检查子节点存在，直接级联删除（应拒绝但实际删除成功） |
| BUG-004 | 低 | 新增节点时排序号字段输入非数字（如字母），前端不做校验直接提交，后端报 500 错误而非友好提示 |

### 用户登录模块（3 个）

| 编号 | 严重等级 | 缺陷描述 |
|------|----------|----------|
| BUG-005 | 中 | 验证码输入错误后，页面清空用户名和密码所有输入框，而非只清空验证码 |
| BUG-006 | 高 | 注册时两次密码不一致仍可提交成功（前端只做了提示但未阻止表单提交，或后端未校验 confirm_password） |
| BUG-007 | 中 | 修改密码时新密码与旧密码相同可以保存成功，后端未校验新旧密码是否相同 |

### 用户管理模块（3 个）

| 编号 | 严重等级 | 缺陷描述 |
|------|----------|----------|
| BUG-008 | 中 | 新增用户手机号输入非 1 开头的数字（如 22123456789）可保存成功，后端不做 1 开头校验 |
| BUG-009 | 中 | 新增用户备注字段输入超过 100 字符可保存成功，后端不做长度校验 |
| BUG-010 | 低 | 用户列表分页，点击最后一页后再点下一页，页码变为超出范围但无数据无提示 |

### 课程管理模块（3 个）

| 编号 | 严重等级 | 缺陷描述 |
|------|----------|----------|
| BUG-011 | 高 | 删除已关联班级的课程时，后端不检查关联关系，直接删除课程，导致班级关联数据变成脏数据 |
| BUG-012 | 中 | 课程名称输入 51 个字符（超过 50 限制）可保存成功，后端不做长度校验 |
| BUG-013 | 中 | 课程描述字段输入超过 200 字符可保存成功 |

### 题库管理模块（7 个，缺陷最多）

| 编号 | 严重等级 | 缺陷描述 |
|------|----------|----------|
| BUG-014 | 严重 | 试题搜索按"题目类型"筛选时，后端查询条件写错（如用 == 比较但字段映射错误），始终返回空列表 |
| BUG-015 | 中 | 添加试题备注字段超过 100 字符时，后端返回"未知错误"而非"备注不能超过 100 个字符"的明确提示 |
| BUG-016 | 严重 | 提交审核操作后，后端不更新试题状态，试题状态仍为"未审核"（update 语句缺失或条件错误） |
| BUG-017 | 严重 | 已被作业引用的试题可以删除，后端不检查引用关系 |
| BUG-018 | 中 | 试题搜索按"所属课程"筛选时，模糊查询的 LIKE 写法错误，只能精确匹配 |
| BUG-019 | 低 | 试题审核驳回时不填写驳回原因也能提交成功，后端不校验驳回原因必填 |
| BUG-020 | 中 | 试题分值输入 0 或 101 时可保存成功，后端不做 1-100 范围校验 |

### 作业管理模块（5 个）

| 编号 | 严重等级 | 缺陷描述 |
|------|----------|----------|
| BUG-021 | 严重 | 作业搜索按钮点击后，前端 JS 事件绑定错误或后端查询条件错误，列表无任何反应 |
| BUG-022 | 高 | 创建作业时截止时间早于发布时间可保存成功，后端不做时间逻辑校验 |
| BUG-023 | 严重 | 学生提交作业后，教师端查看提交状态的查询逻辑错误，始终显示"未提交" |

| BUG-025 | 中 | 删除已发布状态的作业可以成功，后端不检查作业状态是否为"未发布" |
